三种 Demo 方案

选一个最适合你的技术能力和展示目标的方向

A

纯 Claude API — 最简单

输入公司名 + 联系人姓名 + 职位，Claude 基于训练知识生成完整 brief

优点

  • 一个 API key 搞定
  • vibe coding 友好，2-3 天可出
  • 无需部署复杂后端

缺点

  • "近期新闻"可能过时
  • 对冷门公司效果差
B

Claude + 网页搜索 — 推荐 ★

调用 Tavily / Perplexity 搜索 API 获取实时新闻和 LinkedIn 数据，再喂给 Claude 生成 brief

优点

  • 新闻/融资/招聘数据真实准确
  • 展示"AI + 数据整合"能力
  • Tavily 有免费额度够 demo 用
  • 最接近真实产品

缺点

  • 需要两个 API key
  • 稍复杂，需要一个简单后端
C

Claude + 用户粘贴上下文

用户输入公司名 + 联系人后，可选粘贴一段 LinkedIn bio 或公司简介，Claude 结合两者生成

优点

  • 一个 API key，无需搜索服务
  • 用户可控数据质量
  • 比 A 方案输出质量更高

缺点

  • 用户需要手动粘贴内容
  • 增加 demo 摩擦感
  • Portfolio 访客可能不愿意操作

我推荐 B 方案：实时数据让 brief 真实可信，Tavily 免费额度足够，对 portfolio 访客来说"输入公司名就能出真实结果"的体验远比其他方案更有说服力。
